How to adapt an asset

The question is never which tools we have, but what exists that works. Run every asset through three tests before you trust it.

  1. Understand it. Read the prompt. If the procedure is not legible to you, it is not ready.
  2. Decompose it. Strip it to its core function, the part that holds no tool. If you cannot rebuild that core from scratch, you do not own it yet.
  3. Adapt it. Pour in the domain knowledge the source could not have: the break-cause taxonomy, the remediation routing rules, the income-type to tax-form mapping. The domain-gap note on each card states what to supply.

Treat credible repos as specifications. Harvest the decomposition, drop the code that will not run, re-express the rest as prompts.
