# Writing skills harvest candidates

## Recommendation

Keep these folders as source material for the next source-family harvest. They are credible generic writing skills, but they are not ready catalog entries for the tax-ops library.

Do not promote them into `content/entries/` or the plugin marketplace until a tax-ops use case passes the three tests in `docs/north-star.md`: understand the source, decompose the core function, and adapt it to prime-brokerage tax reporting.

## Candidates

- `content-strategy/`: planning pillars, topic clusters, buyer-stage topics, and editorial workflows. Possible tax-ops adaptation: KB coverage planning or recurring operations-report topic planning.
- `copywriting/`: conversion-copy structure and message clarity. Possible tax-ops adaptation: field-status reply quality, client-facing explanation drafts, or leadership update copy.
- `deep-write/`: long-form article research, framing, drafting, rubric scoring, and revision loops. Possible tax-ops adaptation: KB article drafting or post-incident writeups.

## Why this path

`content/entries/` is the live flat YAML source for the offline catalog. These folders use native `SKILL.md` packs with reference files, and the current build skips directories. Keeping them under `content/harvest/` preserves the source material while keeping the build surface honest.
